3. Hosting Sphinx Doc on GitHub Pages

Reference: Deploy Sphinx documentation to GitHub Pages (https://coderefinery.github.io/documentation/gh_workflow/)

To host your docs on GitHub Pages, we need to enable GitHub Pages for our repo. To do that, we need to create a branch that only contains html files. We’ll name this branch gh-pages, as GitHub looks for this name and automatically starts hosting your documentation on pages.

3.1. Creating a github repo

  • Click on Create a repository

  • Goto branch main

  • Create the documentation source folder docs/

  • Create a gh-pages branch

3.2. Add the GitHub Action to the new Git repository

  • Goto a github repo

  • Goto branch main

  • Create an empty file .nojekyll

  • Add a new file at .github/workflows/documentation.yaml, containing:

    Listing 3.1 contents of the .github/workflows/documentation.yaml file
     1name: Publish Sphinx Documentation
 2
 3on: [push, pull_request, workflow_dispatch]
 4
 5permissions:
 6  contents: write
 7
 8jobs:
 9  publish_sphinx_docs:
10    runs-on: ubuntu-latest
11    permissions:
12      contents: write
13    steps:
14      - uses: actions/checkout@v4
15      - uses: actions/setup-python@v5
16        with:
17          python-version: '3.12'
18      - name: Install dependencies
19        run: |
20          pip install sphinx sphinx_rtd_theme myst_parser sphinx-autobuild sphinx-design
21      - name: Sphinx build
22        run: |
23          sphinx-build docs _build
24      - name: Deploy to GitHub Pages
25        uses: peaceiris/actions-gh-pages@v3
26        if: ${{ github.event_name == 'push' && github.ref == 'refs/heads/main' }}
27        with:
28          publish_branch: gh-pages
29          github_token: ${{ secrets.GITHUB_TOKEN }}
30          publish_dir: _build/
31          force_orphan: true

3.3. Enable GitHub Pages

  • On GitHub go to Settings -> Pages.

  • In the Source section, choose Deploy from a branch in the dropdown menu.

  • In the Branch section choose gh-pages and /root in the dropdown menus and click save.

  • Verify the pages deployment in the Actions list. Navigate to the Actions tab, verify:

    • Under All workflows

    • Check documentation workflow is running

    • Click to see the running details

3.4. Verify the result

Github pages should now be live on https://USER.github.io/<repo-name>/ (replace USER).

3.5. Verify refreshing the documentation

  • Commit some changes to the documentation

  • Verify that the documentation website refreshes after changes (can take few seconds or a minute)
